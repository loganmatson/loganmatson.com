01 — Case Study

A Real Impact

8 Years of Epilepsy and Invasive Neurosurgery — now my health is driven by an autonomous system that plans, tracks, and self-corrects.

For years, recovery from seizures meant spending 90% of my time planning and 10% acting — with no real loop back to see what worked. So I built one. Six autonomous agents — meal planner, training calendar, neurologist risk forecaster, Whoop analyst, Google Workspace operator, weekly orchestrator — running on a shared pipeline-state.json nervous system. Twenty-five-plus subagents handle parallel work. Calibration weights self-tune monthly through a retrospective validator that checks every forecast against real biometric outcomes.

Key insights surfaced

  • HRV → next-day recovery r = 0.815 — the strongest predictor in the system
  • Strength days yield +6% next-day recovery vs. run days; rest days redistributed accordingly
  • Forecast-driven rest has cut overtraining flags and lifted average weekly training quality

Why I built this

Eight years of epilepsy taught me that anxiety is what bad planning feels like — I was spending 90% of my energy guessing, and 10% acting on it. The unlock came when the agents started surfacing things I'd never have caught by hand:

  • The neurologist agent flagged hydration drop-offs the day after long runs — a pattern I'd been missing for months that mapped directly to how I felt the next morning.
  • The Whoop analyst found that meditation moved my deep sleep more than any other input — more than caffeine timing, more than screen time, more than bedtime itself.

02 — Architecture

How I Build

A pattern, not a product. Anyone can apply it.

Agents

Autonomous workers that complete a job end-to-end — from input to delivered output, no babysitting.

Skills

Reusable capabilities the agents draw from. Think of them as muscle memory the system never forgets.

Workflows

Orchestrated sequences. One agent's output becomes the next agent's input — a chain of small, validated steps.

Subagent Dispatching

Independent tasks fanned out in parallel. Each runs in its own context so nothing bleeds into the others.

State Files

A shared JSON nervous system. Every component writes its block; every component reads what it needs.

Self-Calibration

Agents that audit their own forecasts and tune their weights from the gap. They get sharper with every run.
